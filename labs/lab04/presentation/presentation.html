Цели и задачи работы

Цель лабораторной работы

Установить и настроить среду GNS3 и виртуальные образы маршрутизаторов для моделирования сетевой инфраструктуры.

Ход выполнения

Запуск виртуальной машины GNS3

Была запущена виртуальная машина GNS3 VM, после чего открыто приложение GNS3.
При первом запуске выбран режим работы Run appliances in a virtual machine.

Настройка подключения контроллера

В мастере настройки указаны параметры подключения:
- Protocol: HTTP
- Host: 192.168.133.131
- Port: 80
- Username: admin

После подтверждения соединение установлено успешно.

Добавление образа FRR

Через меню New template выбран маршрутизатор FRR (FRRouting Project).
В процессе установки использован образ frr-8.2.2.qcow2.

Параметры шаблона FRR

Настроены параметры виртуальной машины:
- RAM: 256 MB
- vCPUs: 1
- Console type: telnet
- On close: Send the shutdown signal (ACPI)

Настройка виртуального диска FRR

Активирована опция Automatically create a config disk on HDD, обеспечивающая сохранение конфигурации устройства.

Проверка запуска FRR

После установки выполнен запуск устройства, в консоли PuTTY отображается баннер FRRouting 8.2.2, что подтверждает успешную загрузку.

Добавление образа VyOS

Аналогичным образом добавлен образ маршрутизатора VyOS.
Выбрана версия VyOS 1.3.3-qemu.

Проверка запуска VyOS

После установки в консоли PuTTY появился приветственный экран VyOS, подтверждающий корректную инициализацию системы.

Выводы

Итог лабораторной работы

В ходе работы были установлены и настроены образы маршрутизаторов FRR и VyOS в среде GNS3.
Оба устройства успешно запущены, что подтверждает готовность виртуальной лабораторной среды для дальнейшего моделирования сетевых топологий.
